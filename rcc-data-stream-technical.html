  • Power BI Pro — visualization, automated dashboard refresh from SharePoint

  • Zera (AI) — automated data pull, report synthesis, M&A scans; depends on clean SP structure

Early Adopter Strategy

Identify 1-2 "bridge people" per department — those who are slightly more tech-comfortable than average. Brief them first, get them comfortable, then let them become peer coaches. Do NOT launch to the full team simultaneously.

Wk 3–4

Create SharePoint folder hierarchy (2-level max). Set permissions per domain. Create file naming cheat sheet (print-ready A5). Brief department heads 1:1. Set up MS Lists for deal pipeline tracking. Owner: BA · Deliverable: SP structure live + dept head signoff · Risk: Permissions misconfiguration
Wk 5–6
Phase 3: Pilot — Property Operations
Pilot with LMSG daily report team (2-3 people). Train on naming + upload. Zera validates daily. Fix any workflow gaps. Stream A briefing distributed. Owner: BA + LMSG ops · Deliverable: 5 consecutive days of clean uploads · Risk: Staff resistance
Wk 7–8
Phase 4: Expand to Investment Pipeline
Migrate active deal files to /Investments/[DealCode]/. Set up MS Lists deal tracker. Bach + BA validate deal memo structure. Zera M&A scan outputs routed to SP. Owner: Bach + BA · Deliverable: All active deals in SP structure · Risk: Incomplete legacy files
Wk 9–10
Phase 5: Financial Data Migration
Move monthly P&L and budget files to SP. BA training on financial data standards. FX exposure tracking in MS Lists. Zera financial report auto-pull tested. Owner: Finance admin + BA · Deliverable: 3 months financial history migrated · Risk: Sensitive data access control

Phase 6: Full Rollout & Retrospective
All departments using SP. Full automation: Zera pulls daily → dashboard → Telegram. Retrospective: what worked, what didn't. SOP finalized and approved. Success metrics reviewed. Owner: BA · Deliverable: SOP v1.0 signed off · Risk: None blocking if phases 1-5 complete

What Breaks Without Data Structure

🔴 Zera Needs (to work reliably)

  • Consistent file naming (finds files by pattern)
  • Fixed folder paths (hardcoded in scripts)
  • One file per date per property (no duplicates)
  • SharePoint Graph API token (auth)
  • Excel sheets with consistent tab names

⚠️ What Causes Failures Today

  • File named "baocao.xlsx" → script can't find it
  • File in wrong folder → Graph API returns 404
  • Two files same date → picks wrong one
  • SharePoint API timeout → dashboard skipped
  • Sheet tab renamed → parsing fails silently

Bottom line: Every hour invested in data structure = multiple hours saved in Zera maintenance and debugging. Clean data architecture is the foundation that makes AI automation reliable vs brittle.
