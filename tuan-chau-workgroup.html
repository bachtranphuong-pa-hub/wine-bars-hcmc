File Versioning & Working Rules
📌 Versioning Rules
  • Tên file: [WorkstreamCode]_[FileType]_v[X]_[YYYY-MM-DD]
  • Không xóa version cũ — archive vào subfolder /archive/
  • Mỗi update phải ghi change log 1 dòng vào MEMORY.md
  • Version lẻ (v1, v3...) = draft | Version chẵn (v2, v4...) = reviewed
  • Output file = đóng băng, không edit — tạo version mới nếu cần sửa
🤖 Zera Orchestrator Rules
  • Mỗi thứ Hai: post Weekly Progress Summary vào Main Channel
  • Khi có update trong topic: cập nhật MEMORY.md của workstream đó
  • Flag cross-workstream dependencies ngay khi detect
  • Sub-personality: adapt tone/skill khi vào từng topic (PM mode, CFO mode...)
  • Không tự quyết định — escalate lên Bach khi cần approval
📂 MEMORY.md Convention
  • Mỗi folder có 1 MEMORY.md riêng — context of that workstream
  • Ghi: current status, last decision, open items, key numbers
  • Update mỗi khi có decision hoặc file version mới
  • 00-Master/MEMORY.md = tổng hợp từ tất cả workstreams
  • Không ghi transcript — chỉ ghi decisions và facts
🔄 Forecast → Management System
  • Forecast file: readonly sau khi project starts — baseline reference
  • Management file: live update, so sánh với forecast mỗi tháng
  • Variance >10% so với forecast → alert vào Main Channel
  • Quarterly: review tất cả files, upgrade stage nếu cần (Forecast → WIP → Output)
  • Project close: toàn bộ archive + Lessons Learned document
